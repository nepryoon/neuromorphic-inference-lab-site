What is proven here

The demo shows retrieval transparency (trace + similarity), grounded drafting from retrieved chunks, and a minimal regression-style evaluation loop. This is the hiring signal: auditability and iteration discipline.

Production design choices

In production: embeddings + vector store, hybrid retrieval, reranking, strict citations policy, refusal on weak retrieval, and schema-validated outputs. CI/CD covers prompts, retrieval configs, and eval sets.

Vector store Hybrid retrieval CI/CD for prompts

Evaluation harness (what recruiters look for)

Metrics: retrieval hit-rate, precision@k, citation correctness, faithfulness, latency, and cost per query. Process: regression prompts + labelled failure cases + replay on every release. Monitoring alerts on drift/regressions.
