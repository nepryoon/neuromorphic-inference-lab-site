Centrico LiveLab — End-to-End MLOps

A production-grade MLOps reference implementation covering the complete lifecycle: data ingestion and preparation, feature engineering, model training with versioning, inference API serving, observability (Prometheus + Grafana), and CI/CD quality gates. Designed to demonstrate full-stack ML engineering discipline.

What problem it solves

Most ML projects stop at "model in a notebook." Centrico LiveLab demonstrates production operationalization: repeatable data pipelines, training orchestration, versioned artefacts, live serving, monitoring infrastructure, and automated quality checks in CI.

Business impact

  • End-to-end visibility: from raw data ingestion to inference results.
  • Reliable deployment pipeline with quality gates (prevents broken models from reaching production).
  • Monitoring and alerting infrastructure for operational confidence.
  • Reproducible training runs with versioned artefacts and lineage tracking.

System architecture

  • Data layer: ingestion pipelines with validation and schema enforcement.
  • Training layer: orchestrated training with experiment tracking and model registry.
  • Serving layer: FastAPI inference endpoint with health checks and metrics.
  • Observability: Prometheus metrics collection + Grafana dashboards.
  • CI/CD: automated tests, model evaluation gates, and deployment workflows.

System components

The system is structured as modular components, each with clear responsibilities and interfaces. This separation enables independent development, testing, and deployment of each layer.

1. Data Engineering Pipeline

Ingestion from multiple sources (CSV, databases, APIs) with validation, cleaning, and transformation logic. Schema enforcement ensures data quality at ingestion time. Feature stores enable consistent feature generation across training and inference.

Schema Validation Data Quality Checks Feature Store Pipeline Orchestration

2. Training & Model Management

Reproducible training pipeline with experiment tracking (MLflow), hyperparameter tuning, and model registry. Each training run produces versioned artefacts (model binaries, metrics, metadata) that are stored and tracked for lineage and rollback capabilities.

MLflow Tracking Model Registry Versioned Artefacts Hyperparameter Tuning

3. Inference API & Serving

FastAPI-based inference service with standardized request/response schemas, health endpoints, and readiness probes. Containerized for consistent deployment across environments. Supports batch and real-time inference patterns.

FastAPI Docker Health Checks Batch & Real-time

4. Monitoring & Observability

Prometheus scrapes metrics from the inference API (request counts, latencies, error rates, model predictions distribution). Grafana dashboards visualize system health, model performance drift indicators, and infrastructure metrics. Alerting rules trigger notifications for anomalies.

Prometheus Grafana Alerting Drift Detection

5. CI/CD & Quality Gates

Automated CI pipeline validates code quality (linting, type checking), runs unit and integration tests, evaluates model performance on hold-out data, and enforces quality gates before promotion to production. Deployment workflows support blue-green and canary deployment strategies.

GitHub Actions Quality Gates Automated Testing Deployment Strategies

Local setup & verification

The system can be run entirely locally using Docker Compose, enabling developers to test and verify the complete stack without cloud dependencies.

Quick start

  1. Clone the repository: git clone https://github.com/nepryoon/centrico-livelab-mlops
  2. Run setup: docker-compose up
  3. Access services:
    • API docs: http://localhost:8000/docs
    • Grafana: http://localhost:3000
    • Prometheus: http://localhost:9090

Verification checklist

  • ✓ Data pipeline runs without errors
  • ✓ Training produces versioned model artefacts
  • ✓ Inference API responds to health checks
  • ✓ Prometheus collects metrics from API
  • ✓ Grafana dashboards display system metrics
  • ✓ CI pipeline passes all quality gates
