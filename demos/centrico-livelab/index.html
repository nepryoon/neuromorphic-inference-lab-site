Architecture overview

The system demonstrates end-to-end thinking:

  • Data Ingestion: Fetches real-time data from CityBikes and Open-Meteo APIs into PostgreSQL (raw JSONB storage)
  • Feature Engineering: Pandas-based transformation pipeline for structured feature extraction
  • Training: Docker-based trainer that reads from DB, trains scikit-learn pipeline, outputs versioned artifacts
  • Inference API: FastAPI service loading model.joblib + metadata.json, serving predictions via /predict
  • Monitoring: Prometheus metrics exposure (/metrics) with Grafana dashboards
  • CI Pipeline: GitHub Actions with unit/contract tests + synthetic training + model quality gate (F1 >= 0.60)

System architecture

Complete MLOps delivery loop demonstrating production-ready engineering discipline.

Data flow

Public APIs (CityBikes + Open-Meteo) → Ingestion Container → PostgreSQL (raw JSONB) → Trainer Container (scikit-learn + feature engineering) → Inference API (FastAPI: /predict, /health, /metrics, /explain) → Prometheus → Grafana

CI/CD: GitHub Actions with quality gate (F1 ≥ 0.60) → AWS ECS Fargate (via OIDC, no long-lived credentials) → S3 (versioned model artefacts) + ECR (Docker images)

Key design decisions

Quality & reliability

  • Adaptive quantile threshold: Avoids single-class training failures when data is imbalanced
  • Model quality gate: F1 ≥ 0.60 enforced in CI before any deployment
  • Prometheus metrics: Every /predict call emits latency + model_loaded + version
  • LLM /explain endpoint: gpt-4o-mini with graceful fallback when key not set

Infrastructure & security

  • AWS deployment via GitHub OIDC: No long-lived credentials
  • Terraform IaC: Full infra (VPC, ECS, ALB, ECR, S3, IAM, SSM, CloudWatch)
  • Target: AWS account 102724112773, region eu-south-1
  • Secrets management: SSM Parameter Store
